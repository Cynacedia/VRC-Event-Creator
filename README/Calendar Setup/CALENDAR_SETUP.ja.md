# カレンダー連携セットアップガイド

このガイドでは、VRC Event Creatorにおけるカレンダーファイル（.ics）生成、Discord Webhook投稿、Discordスケジュールイベントの設定手順を説明します。これら3つの機能は完全に独立しています — ワークフローに合わせて自由に組み合わせて使用できます。

---

## 概要

VRC Event Creatorでは、VRChatイベントを作成または自動化する際に3つのアクションを利用できます。各アクションはテンプレートごと、イベントごとに独立して切り替えられます：

- **「.icsカレンダー招待を作成」** — オプションのリマインダー付きの標準`.ics`カレンダーファイルを生成し、ローカルディレクトリに自動保存
- **「Discord Webhookに投稿」** — Webhookを通じてDiscordチャンネルにアナウンスを投稿（カレンダーも有効な場合は`.ics`添付可能）
- **「Discordイベントを作成」** — BotによりDiscordサーバーにスケジュールイベントを作成

複数の機能を有効にすると、自然に連携します：

| Discordイベント | Webhook | カレンダー (.ics) | 動作 |
|---|---|---|---|
| オン | オフ | オフ | Discordスケジュールイベントのみ作成 |
| オフ | オン | オフ | Webhookがイベント詳細のEmbedを投稿 |
| オフ | オフ | オン | `.ics`ファイルがローカルディレクトリに自動保存 |
| オン | オン | オフ | Discordイベント作成 + Webhookがイベントリンクを投稿 |
| オン | オフ | オン | Discordイベント作成 + `.ics`自動保存 |
| オフ | オン | オン | WebhookがEmbed + `.ics`添付で投稿、ローカルにも保存 |
| オン | オン | オン | Discordイベント + Webhookがイベントリンク + `.ics`添付 + 自動保存 |

## ステップ1：カレンダーファイル生成を有効にする

1. **設定** > **詳細設定**を開く
2. **「カレンダーファイル生成を有効にする」**にチェック

これにより、テンプレートとイベント作成時に**「.icsカレンダー招待を作成」**トグルが使用可能になります。

### 保存ディレクトリ

カレンダーファイル生成が有効な場合、`.ics`ファイルは常にローカルディレクトリに保存されます。デフォルトの保存先は`Documents/VRC Event Creator .ics/`で、初回保存時に作成されます。

ファイルは`{ディレクトリ}/{グループ名}/{イベント名 - 日付}.ics`として保存されます。保存先を変更するには、**設定** > **アプリ情報**の**カレンダー保存ディレクトリ**横にある**変更**ボタンを使用してください。

---

## ステップ2：Discord Webhookを設定する（オプション）

Webhookは特定のDiscordチャンネルにアナウンスを投稿します。カレンダーファイルやDiscordイベントとは独立した機能で、どちらとも組み合わせて、あるいは単独で使用できます。

1. Discordで、アナウンスを投稿したいチャンネルを右クリック
2. **チャンネルの編集** > **連携サービス** > **ウェブフック** > **新しいウェブフック**をクリック
3. WebhookのURLをコピー
4. VRC Event Creatorで、**設定** > **Discord連携** > グループを選択
5. **「Webhookを有効化」**にチェックし、WebhookのURLを貼り付け
6. **Webhookをテスト**をクリックして確認し、**保存**をクリック

Webhookとカレンダーの両方がイベントで有効な場合、`.ics`ファイルがWebhook投稿に添付されます。Webhookのみ有効（カレンダー無効）の場合は、イベント詳細のEmbedのみ投稿されます（`.ics`添付なし）。

Discordスケジュールイベントも作成された場合、WebhookメッセージにはEmbedの代わりにDiscordイベントリンクが含まれます。

---

## ステップ3：テンプレートを設定する

1. **テンプレート管理**を開き、テンプレートを編集（または作成）
2. **基本**タブに、設定状況に応じて最大3つの投稿トグルが表示されます：
   - **「.icsカレンダー招待を作成」** — カレンダーファイル生成が有効な場合に表示
   - **「Discordイベントを作成」** — Discord Botがグループに設定されている場合に表示
   - **「Discord Webhookに投稿」** — Webhook URLがグループに設定されている場合に表示
3. このテンプレートで使用したいものを有効にする
4. カレンダーが有効な場合、**スケジュール**タブに**「.icsカレンダーリマインダー」**カードが表示される
5. **「.icsカレンダーリマインダーを有効にする」**にチェックし、お好みのリマインダー間隔を追加
6. テンプレートを保存

リマインダーは主要なカレンダーアプリすべてに対応したプリセット間隔を使用します：5分、10分、15分、30分、1時間、2時間、4時間、8時間、12時間、1日、2日、1週間。

> **注意：** 一部のカレンダーアプリ（Outlookなど）は最初のリマインダーのみを使用します。互換性を最大化するため、最も長いリマインダーが先頭に配置されます。Google Calendarはインポート時にカスタムリマインダーを無視し、デフォルトの通知設定を使用します。

---

## ステップ4：イベントを作成する

イベントを作成する際（手動または自動化）：

- **日付**のステップに**「.icsカレンダー招待を作成」**が表示されます（テンプレートから継承、上書き可能）
- その下の**「.icsカレンダーリマインダーを有効にする」**でイベントごとにリマインダーをカスタマイズできます
- **詳細**のステップに**「Discordイベントを作成」**と**「Discord Webhookに投稿」**が別々のトグルとして表示されます
- テンプレートのすべての設定はイベントごとに上書き可能です

## よくある質問

### どのカレンダーアプリが.icsファイルに対応していますか？

主要なカレンダーアプリすべて：Outlook、Apple Calendar、Google Calendar、Thunderbird、およびiCalendar標準に対応するすべてのアプリ。

### リマインダーはすべてのカレンダーアプリで機能しますか？

複数のリマインダーはApple CalendarとThunderbirdで機能します。Outlookは最初のリマインダーのみ使用します。Google Calendarはインポート時にリマインダーを完全に無視します。

### カレンダーファイルなしでWebhookを使用できますか？

はい。Webhookはカレンダーファイル生成が無効な場合でも、イベント詳細のEmbedを投稿します。「.icsカレンダー招待を作成」を有効にせずに、テンプレートで「Discord Webhookに投稿」を有効にしてください。

### Discordイベント作成なしでWebhookを使用できますか？

はい。Webhook、Discordイベント、カレンダーファイルは完全に独立しています。どの組み合わせでも動作します。

### WebhookのURLは機密ですか？

はい — WebhookのURLを知っている人は誰でもそのチャンネルにメッセージを送信できます。パスワードと同様に扱ってください。OSの安全なストレージを使用して暗号化され、ローカルに保存されます。

## トラブルシューティング

| 問題 | 解決方法 |
|---|---|
| .icsファイルが生成されない | 詳細設定で「カレンダーファイル生成を有効にする」がオンになっていること、テンプレートまたはイベントで「.icsカレンダー招待を作成」がチェックされていることを確認 |
| Webhookが投稿しない | Discord設定の「Webhookをテスト」でWebhook URLを確認。グループで「Webhookを有効化」がオンになっていること、テンプレートで「Discord Webhookに投稿」がチェックされていることを確認 |
| Webhookが投稿するが.icsが添付されない | イベントで「.icsカレンダー招待を作成」も有効にする必要があります。これがないと、WebhookはEmbedまたはイベントリンクのみ投稿します |
| Outlookでリマインダーが機能しない | Outlookは最初のリマインダーのみサポート。互換性のため、アプリは最も長いリマインダーを先頭に配置 |
| Google Calendarでリマインダーが機能しない | Google Calendarは.icsインポート時にカスタムリマインダーを無視。インポート後に手動でリマインダーを設定 |
| ファイルが間違った場所に保存される | ファイルは`{保存先}/{グループ名}/`に保存されます。デフォルトは`Documents/VRC Event Creator .ics/`です。設定 > アプリ情報で変更可能 |
