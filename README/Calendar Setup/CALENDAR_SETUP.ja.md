# カレンダー連携セットアップガイド

VRC Event Creatorのカレンダーファイル（.ics）生成およびDiscord Webhook配信を設定する手順です。設定が完了すると、VRChatイベントを作成する際にカレンダー招待ファイルが自動生成され、オプションでDiscordチャンネルに投稿できます。

---

## 概要

カレンダー連携は、Outlook、Apple Calendar、Google Calendarなどのカレンダーアプリにインポート可能な標準`.ics`カレンダーファイルを作成します。これらのファイルにはイベントの詳細とオプションのリマインダーが含まれます。

2つの配信方法があります（イベントごとにどちらか一方のみ）：

- **Discord Webhook** — `.ics`ファイルをDiscordチャンネルにイベントのEmbedまたはDiscordイベントリンク付きで投稿
- **自動保存** — `.ics`ファイルをローカルディレクトリに自動保存

Discord Webhookが設定されており、イベントがDiscordに投稿する設定になっている場合はWebhookが使用されます。それ以外の場合は、設定されたローカルディレクトリにファイルが保存されます。

---

## ステップ1：カレンダーファイル生成を有効にする

1. **設定** > **一般**を開く
2. **「カレンダーファイル生成を有効にする」**にチェック

これにより、テンプレートやイベント作成時にカレンダーオプションが利用可能になります。

## ステップ2：配信方法を設定する

### オプションA：Discord Webhook（推奨）

Webhookはカレンダーファイルを特定のDiscordチャンネルに投稿します。Webhook自体にBotは不要です。

1. Discordで、カレンダーファイルを投稿したいチャンネルを右クリック
2. **チャンネルの編集** > **連携サービス** > **ウェブフック** > **新しいウェブフック**をクリック
3. WebhookのURLをコピー
4. VRC Event Creatorで、**設定** > **Discord連携** > グループを選択
5. **「.icsをDiscordに投稿」**にチェックし、WebhookのURLを貼り付け
6. **「Webhookをテスト」**をクリックして接続を確認し、**「保存」**をクリック

Discord イベント作成（Botトークン）も設定済みの場合、Webhookは独立したEmbedの代わりにDiscordイベントへのリンクを投稿します。`.ics`ファイルはどちらの場合も添付されます。

### オプションB：ローカルディレクトリへの自動保存

1. **設定** > **アプリ情報**で**カレンダー保存ディレクトリ**を見つける
2. **「デフォルトを作成」**（`Documents/VRC Event Creator .ics/`を作成）または**「参照」**をクリックして任意の場所を選択
3. ファイルは自動的に`{ディレクトリ}/{グループ名}/{イベント名 - 日付}.ics`に保存されます

---

## ステップ3：テンプレートを設定する

1. **テンプレート管理**を開き、テンプレートを編集（または作成）
2. **基本**タブで**「.icsカレンダー招待を作成」**にチェック
3. **スケジュール**タブに新しいカード**「.icsカレンダーリマインダー」**が表示される
4. **「.icsカレンダーリマインダーを有効にする」**にチェックし、お好みのリマインダー間隔を追加
5. テンプレートを保存

リマインダーは主要なカレンダーアプリすべてに対応したプリセット間隔を使用します：5分、10分、15分、30分、1時間、2時間、4時間、8時間、12時間、1日、2日、1週間。

> **注意：** 一部のカレンダーアプリ（Outlookなど）は最初のリマインダーのみを使用します。互換性を最大化するため、最も長いリマインダーが先頭に配置されます。Google Calendarはインポート時にカスタムリマインダーを無視し、デフォルトの通知設定を使用します。

---

## ステップ4：イベントを作成する

イベントを作成する際（手動または自動化）：

- **日付**のステップに**「.icsカレンダー招待を作成」**トグルが表示されます（選択したテンプレートから継承、または手動で設定可能）
- その下の**「.icsカレンダーリマインダーを有効にする」**でイベントごとにリマインダーをカスタマイズできます
- **詳細**のステップに表示される**「Discordに投稿」**がDiscordイベントとWebhook配信の両方を制御します

テンプレートのすべての設定はイベントごとに上書き可能です。

---

## 連携の仕組み

| Discordイベント | Webhook | カレンダー | イベント作成時の動作 |
|---|---|---|---|
| 有効 + 設定済み | 設定済み | 有効 | Discordイベント作成、Webhookがイベントリンク + .icsを投稿 |
| 無効または未設定 | 設定済み | 有効 | Webhookがイベント詳細のEmbed + .icsを投稿 |
| 任意 | 未設定 | 有効 + 保存先設定済み | .icsファイルがローカルディレクトリに自動保存 |
| 任意 | 未設定 | 有効 + 保存先未設定 | .ics配信なし（手動保存のみ） |

## よくある質問

### どのカレンダーアプリが.icsファイルに対応していますか？

主要なカレンダーアプリすべて：Outlook、Apple Calendar、Google Calendar、Thunderbird、およびiCalendar標準に対応するすべてのアプリ。

### リマインダーはすべてのカレンダーアプリで機能しますか？

複数のリマインダーはApple CalendarとThunderbirdで機能します。Outlookは最初のリマインダーのみ使用します。Google Calendarはインポート時にリマインダーを完全に無視します。

### Discordイベント作成なしでWebhookを使用できますか？

はい。WebhookとBotトークンは独立した機能です。Discord Botを設定せずに、Webhookでカレンダーを配信できます。

### WebhookのURLは機密ですか？

はい — WebhookのURLを知っている人は誰でもそのチャンネルにメッセージを送信できます。パスワードと同様に扱ってください。OSの安全なストレージを使用して暗号化され、ローカルに保存されます。

## トラブルシューティング

| 問題 | 解決方法 |
|---|---|
| .icsファイルが生成されない | 設定 > 一般で「カレンダーファイル生成を有効にする」がオンになっていること、テンプレートまたはイベントで「.icsカレンダー招待を作成」がチェックされていることを確認 |
| Webhookが投稿しない | Discord設定の「Webhookをテスト」でWebhook URLを確認。グループで「.icsをDiscordに投稿」が有効であることを確認 |
| Outlookでリマインダーが機能しない | Outlookは最初のリマインダーのみサポート。互換性のため、アプリは最も長いリマインダーを先頭に配置 |
| Google Calendarでリマインダーが機能しない | Google Calendarは.icsインポート時にカスタムリマインダーを無視。インポート後に手動でリマインダーを設定 |
| 自動保存が機能しない | 設定 > アプリ情報で保存ディレクトリが設定されていることを確認 |
| ファイルが間違った場所に保存される | ファイルは`{保存先}/{グループ名}/`に保存されます。グループ名とディレクトリパスを確認 |
