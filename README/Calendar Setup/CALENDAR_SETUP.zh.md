# 日历集成设置指南

本指南将引导你完成 VRC Event Creator 的日历文件（.ics）生成、Discord Webhook 发布和 Discord 预定活动的设置。这三项功能完全独立 — 可以根据你的工作流程启用任意组合。

---

## 概述

VRC Event Creator 在你创建或自动化 VRChat 活动时提供三项后续操作。每项操作可以按模板和按活动独立切换：

- **"创建.ics日历邀请"** — 生成带有可选提醒的标准 `.ics` 日历文件，自动保存到本地目录
- **"发送 Discord Webhook"** — 通过 Webhook 向 Discord 频道发布公告（如果日历也已启用，可附加 `.ics` 文件）
- **"创建 Discord 活动"** — 通过机器人在你的 Discord 服务器中创建预定活动

启用多项功能时，它们会自然配合：

| Discord 活动 | Webhook | 日历 (.ics) | 行为 |
|---|---|---|---|
| 开 | 关 | 关 | 仅创建 Discord 预定活动 |
| 关 | 开 | 关 | Webhook 发布包含活动详情的嵌入卡片 |
| 关 | 关 | 开 | `.ics` 文件自动保存到本地目录 |
| 开 | 开 | 关 | 创建 Discord 活动 + Webhook 发布活动链接 |
| 开 | 关 | 开 | 创建 Discord 活动 + `.ics` 自动保存 |
| 关 | 开 | 开 | Webhook 发布嵌入卡片 + `.ics` 附件，同时自动保存 |
| 开 | 开 | 开 | Discord 活动 + Webhook 发布活动链接 + `.ics` 附件 + 自动保存 |

---

## 第 1 步：启用日历文件生成

1. 打开 **设置** > **高级设置**
2. 勾选 **"启用日历文件生成"**

这将使 **"创建.ics日历邀请"** 开关在模板和活动创建中可用。

### 保存目录

启用日历文件生成后，`.ics` 文件始终会保存到本地目录。默认位置为 `Documents/VRC Event Creator .ics/`，在首次保存时创建。

文件保存为 `{目录}/{群组名称}/{活动名称 - 日期}.ics`。要更改保存位置，请使用 **设置** > **应用信息** 中 **日历保存目录** 旁边的 **更改** 按钮。

---

## 第 2 步：配置 Discord Webhook（可选）

Webhook 可以向指定的 Discord 频道发布公告。它独立于日历文件和 Discord 活动 — 你可以单独使用，也可以与它们配合使用。

1. 在 Discord 中，右键点击你想要发布公告的频道
2. 点击 **编辑频道** > **集成** > **Webhook** > **新建 Webhook**
3. 复制 Webhook URL
4. 在 VRC Event Creator 中，前往 **设置** > **Discord 集成** > 选择你的群组
5. 勾选 **"启用 Webhook"** 并粘贴 Webhook URL
6. 点击 **测试Webhook** 验证连接，然后点击 **保存**

当 Webhook 和日历同时为某个活动启用时，`.ics` 文件会附加到 Webhook 帖子中。当仅启用 Webhook（未启用日历）时，Webhook 发布包含活动详情的嵌入卡片，不附加 `.ics` 文件。

如果同时创建了 Discord 预定活动，Webhook 消息将包含 Discord 活动链接而非嵌入卡片。

---

## 第 3 步：配置模板

1. 前往 **管理模板** 并编辑（或创建）一个模板
2. 在 **基础** 标签页中，你将看到最多三个发布开关（取决于配置情况）：
   - **"创建.ics日历邀请"** — 当日历文件生成已启用时显示
   - **"创建 Discord 活动"** — 当群组已配置 Discord 机器人时显示
   - **"发送 Discord Webhook"** — 当群组已配置 Webhook URL 时显示
3. 为此模板启用你需要的选项
4. 如果日历已启用，**计划** 标签页会显示 **".ics日历提醒"** 卡片
5. 勾选 **"启用.ics日历提醒"** 并添加你想要的提醒间隔
6. 保存模板

提醒使用与所有主流日历应用兼容的预设间隔：5 分钟、10 分钟、15 分钟、30 分钟、1 小时、2 小时、4 小时、8 小时、12 小时、1 天、2 天、1 周。

> **注意：** 部分日历应用（如 Outlook）只使用第一个提醒。为了最佳兼容性，最长的提醒会被放在最前面。Google Calendar 在导入时会忽略自定义提醒，而使用你的默认通知设置。

---

## 第 4 步：创建活动

创建活动时（手动或通过自动化）：

- **日期** 步骤显示 **"创建.ics日历邀请"**（继承自模板，可覆盖）
- 下方的 **"启用.ics日历提醒"** 允许你按活动自定义提醒
- **详情** 步骤显示 **"创建 Discord 活动"** 和 **"发送 Discord Webhook"** 作为独立的开关
- 模板中的所有设置都可以按活动单独覆盖

## 常见问题

### 哪些日历应用支持 .ics 文件？

所有主流日历应用：Outlook、Apple Calendar、Google Calendar、Thunderbird 以及任何支持 iCalendar 标准的应用。

### 提醒在所有日历应用中都有效吗？

多个提醒在 Apple Calendar 和 Thunderbird 中有效。Outlook 只使用第一个提醒。Google Calendar 在导入时完全忽略提醒。

### 可以不生成日历文件而只使用 Webhook 吗？

可以。即使日历文件生成已禁用，Webhook 也会发布包含活动详情的嵌入卡片。在模板中启用"发送 Discord Webhook"，而不启用"创建.ics日历邀请"即可。

### 可以不创建 Discord 活动而只使用 Webhook 吗？

可以。Webhook、Discord 活动和日历文件完全独立。任何组合都可以使用。

### Webhook URL 是否敏感？

是的 — 任何拥有 Webhook URL 的人都可以向该频道发送消息。请像对待密码一样对待它。它通过操作系统的安全存储进行加密，并存储在本地。

## 故障排除

| 问题 | 解决方案 |
|---|---|
| 未生成 .ics 文件 | 检查高级设置中"启用日历文件生成"是否已开启，以及模板或活动中"创建.ics日历邀请"是否已勾选 |
| Webhook 未发布 | 在 Discord 设置中使用"测试Webhook"验证 Webhook URL。检查该群组是否已启用"启用 Webhook"，以及模板中是否已勾选"发送 Discord Webhook" |
| Webhook 发布了但没有附加 .ics | 活动中还需启用"创建.ics日历邀请"。没有启用时，Webhook 只发布嵌入卡片或活动链接 |
| Outlook 中提醒不起作用 | Outlook 仅支持第一个提醒。应用会将最长的提醒放在最前面以确保兼容性 |
| Google Calendar 中提醒不起作用 | Google Calendar 在 .ics 导入时忽略自定义提醒。导入后请手动设置提醒 |
| 文件保存到了错误的位置 | 文件保存在 `{保存目录}/{群组名称}/`。默认为 `Documents/VRC Event Creator .ics/`。可通过设置 > 应用信息更改 |
