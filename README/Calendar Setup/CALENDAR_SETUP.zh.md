# 日历集成设置指南

本指南将引导你完成 VRC Event Creator 的日历文件（.ics）生成及 Discord Webhook 投递的设置。配置完成后，创建 VRChat 活动时可以自动生成日历邀请文件，并可选择发布到 Discord 频道。

---

## 概述

日历集成会创建标准的 `.ics` 日历文件，可导入 Outlook、Apple Calendar、Google Calendar 及其他日历应用。这些文件包含活动详情和可选的提醒。

有两种投递方式（每个活动只能使用其中一种）：

- **Discord Webhook** — 将 `.ics` 文件发布到 Discord 频道，附带活动嵌入卡片或 Discord 活动链接
- **自动保存** — 将 `.ics` 文件自动保存到本地目录

如果已配置 Discord Webhook 且活动设置为发布到 Discord，则使用 Webhook。否则，文件将保存到配置的本地目录。

---

## 第 1 步：启用日历文件生成

1. 打开 **设置** > **常规**
2. 勾选 **"启用日历文件生成"**

这将使日历选项在模板和活动创建中可用。

## 第 2 步：配置投递方式

### 选项 A：Discord Webhook（推荐）

Webhook 会将日历文件发布到指定的 Discord 频道。Webhook 本身不需要机器人。

1. 在 Discord 中，右键点击你想要发布日历文件的频道
2. 点击 **编辑频道** > **集成** > **Webhook** > **新建 Webhook**
3. 复制 Webhook URL
4. 在 VRC Event Creator 中，前往 **设置** > **Discord 集成** > 选择你的群组
5. 勾选 **"发布.ics到Discord"** 并粘贴 Webhook URL
6. 点击 **"测试Webhook"** 验证连接，然后点击 **"保存"**

如果你还设置了 Discord 活动创建（机器人令牌），Webhook 将发布 Discord 活动链接而非独立的嵌入卡片。`.ics` 文件在两种情况下都会被附加。

### 选项 B：自动保存到本地目录

未配置 Webhook 时，`.ics` 文件会自动保存到本地目录。默认位置为 `Documents/VRC Event Creator .ics/`，在首次保存时创建。

文件保存为 `{目录}/{群组名称}/{活动名称 - 日期}.ics`。要更改保存位置，请使用 **设置** > **应用信息** 中 **日历保存目录** 旁边的 **更改** 按钮。

---

## 第 3 步：配置模板

1. 前往 **管理模板** 并编辑（或创建）一个模板
2. 在 **基础** 标签页中，勾选 **"创建.ics日历邀请"**
3. 在 **计划** 标签页中，会出现一个新的 **".ics日历提醒"** 卡片
4. 勾选 **"启用.ics日历提醒"** 并添加你想要的提醒间隔
5. 保存模板

提醒使用与所有主流日历应用兼容的预设间隔：5 分钟、10 分钟、15 分钟、30 分钟、1 小时、2 小时、4 小时、8 小时、12 小时、1 天、2 天、1 周。

> **注意：** 部分日历应用（如 Outlook）只使用第一个提醒。为了最佳兼容性，最长的提醒会被放在最前面。Google Calendar 在导入时会忽略自定义提醒，而使用你的默认通知设置。

---

## 第 4 步：创建活动

创建活动时（手动或通过自动化）：

- **日期** 步骤显示 **"创建.ics日历邀请"** 开关（继承自所选模板，也可手动配置）
- 下方的 **"启用.ics日历提醒"** 允许你按活动自定义提醒
- **详情** 步骤显示 **"发布到Discord"**，同时控制 Discord 活动和 Webhook 投递

模板中的所有设置都可以按活动单独覆盖。

---

## 功能协作方式

| Discord 活动 | Webhook | 日历 | 创建活动时的行为 |
|---|---|---|---|
| 已启用 + 已配置 | 已配置 | 已启用 | 创建 Discord 活动，Webhook 发布活动链接 + .ics |
| 已禁用或未配置 | 已配置 | 已启用 | Webhook 发布包含活动详情的嵌入卡片 + .ics |
| 任意 | 未配置 | 已启用 | .ics 文件自动保存到本地目录 |

---

## 常见问题

### 哪些日历应用支持 .ics 文件？

所有主流日历应用：Outlook、Apple Calendar、Google Calendar、Thunderbird 以及任何支持 iCalendar 标准的应用。

### 提醒在所有日历应用中都有效吗？

多个提醒在 Apple Calendar 和 Thunderbird 中有效。Outlook 只使用第一个提醒。Google Calendar 在导入时完全忽略提醒。

### 可以不创建 Discord 活动而只使用 Webhook 吗？

可以。Webhook 和机器人令牌是独立的功能。你可以在不设置 Discord 机器人的情况下使用 Webhook 投递日历。

### Webhook URL 是否敏感？

是的 — 任何拥有 Webhook URL 的人都可以向该频道发送消息。请像对待密码一样对待它。它通过操作系统的安全存储进行加密，并存储在本地。

---

## 故障排除

| 问题 | 解决方案 |
|---|---|
| 未生成 .ics 文件 | 检查设置 > 常规中"启用日历文件生成"是否已开启，以及模板或活动中"创建.ics日历邀请"是否已勾选 |
| Webhook 未发布 | 在 Discord 设置中使用"测试Webhook"验证 Webhook URL。检查该群组是否已启用"发布.ics到Discord" |
| Outlook 中提醒不起作用 | Outlook 仅支持第一个提醒。应用会将最长的提醒放在最前面以确保兼容性 |
| Google Calendar 中提醒不起作用 | Google Calendar 在 .ics 导入时忽略自定义提醒。导入后请手动设置提醒 |
| 文件保存到了错误的位置 | 文件保存在 `{保存目录}/{群组名称}/`。默认为 `Documents/VRC Event Creator .ics/`。可通过设置 > 应用信息更改 |
