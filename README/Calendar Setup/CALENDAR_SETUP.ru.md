# Руководство по настройке интеграции с календарём

Это руководство поможет вам настроить генерацию файлов календаря (.ics) и доставку через Discord-вебхук для VRC Event Creator. После настройки при создании мероприятия VRChat автоматически генерируется файл приглашения в календарь, который при желании можно отправить в канал Discord.

---

## Обзор

Интеграция с календарём создаёт стандартные файлы `.ics`, которые можно импортировать в Outlook, Apple Calendar, Google Calendar и другие календарные приложения. Эти файлы содержат информацию о мероприятии и необязательные напоминания.

Существует два способа доставки (для каждого мероприятия — только один):

- **Discord-вебхук** — Публикует файл `.ics` в канале Discord с карточкой мероприятия или ссылкой на мероприятие Discord
- **Автосохранение** — Сохраняет файл `.ics` в локальную директорию автоматически

Если настроен Discord-вебхук и мероприятие помечено для публикации в Discord, используется вебхук. В противном случае файлы сохраняются в указанную локальную директорию.

---

## Шаг 1: Включить создание файлов календаря

1. Откройте **Настройки** > **Общие**
2. Отметьте **«Включить создание файлов календаря»**

После этого параметры календаря станут доступны в шаблонах и при создании мероприятий.

## Шаг 2: Настроить способ доставки

### Вариант A: Discord-вебхук (рекомендуется)

Вебхук публикует файл календаря в определённый канал Discord. Для самого вебхука бот не требуется.

1. В Discord нажмите правой кнопкой мыши на канал, куда хотите отправлять файлы календаря
2. Нажмите **Редактировать канал** > **Интеграции** > **Вебхуки** > **Новый вебхук**
3. Скопируйте URL вебхука
4. В VRC Event Creator перейдите в **Настройки** > **Интеграция с Discord** > выберите свою группу
5. Отметьте **«Отправить .ics в Discord»** и вставьте URL вебхука
6. Нажмите **«Проверить вебхук»**, чтобы убедиться в работоспособности, затем **«Сохранить»**

Если у вас также настроено создание мероприятий Discord (токен бота), вебхук опубликует ссылку на мероприятие Discord вместо отдельной карточки. Файл `.ics` прикрепляется в обоих случаях.

### Вариант B: Автосохранение в локальную директорию

1. В **Настройки** > **Информация о приложении** найдите **Директория сохранения календаря**
2. Нажмите **«Создать по умолчанию»** (создаёт `Documents/VRC Event Creator .ics/`) или **«Обзор»**, чтобы выбрать пользовательское расположение
3. Файлы сохраняются автоматически как `{директория}/{Название группы}/{Название мероприятия - Дата}.ics`

---

## Шаг 3: Настроить шаблоны

1. Перейдите в **Управление шаблонами** и отредактируйте (или создайте) шаблон
2. Во вкладке **Основы** отметьте **«Создать приглашение календаря .ics»**
3. Во вкладке **Расписание** появится новая карточка **«Напоминания календаря .ics»**
4. Отметьте **«Включить напоминания календаря .ics»** и добавьте нужные интервалы напоминаний
5. Сохраните шаблон

Напоминания используют предустановленные интервалы, совместимые со всеми основными календарными приложениями: 5 мин, 10 мин, 15 мин, 30 мин, 1 час, 2 часа, 4 часа, 8 часов, 12 часов, 1 день, 2 дня, 1 неделя.

> **Примечание:** Некоторые календарные приложения (например, Outlook) используют только первое напоминание. Для лучшей совместимости самое длительное напоминание размещается первым. Google Calendar игнорирует пользовательские напоминания при импорте и использует вместо этого ваши настройки уведомлений по умолчанию.

---

## Шаг 4: Создание мероприятий

При создании мероприятия (вручную или через автоматизацию):

- На шаге **Дата** отображается переключатель **«Создать приглашение календаря .ics»** (наследуется из выбранного шаблона или настраивается вручную)
- Ниже **«Включить напоминания календаря .ics»** позволяет настроить напоминания для каждого мероприятия
- На шаге **Детали** отображается **«Опубликовать в Discord»**, который управляет и мероприятием Discord, и доставкой через вебхук

Все настройки шаблона можно изменить для каждого мероприятия.

## Как всё работает вместе

| Мероприятия Discord | Вебхук | Календарь | Что происходит при создании мероприятия |
|---|---|---|---|
| Включены + настроены | Настроен | Включён | Создаётся мероприятие Discord, вебхук публикует ссылку + .ics |
| Отключены или не настроены | Настроен | Включён | Вебхук публикует карточку с деталями мероприятия + .ics |
| Любой | Не настроен | Включён + директория задана | Файл .ics автоматически сохраняется в локальную директорию |
| Любой | Не настроен | Включён + директория не задана | Доставка .ics отсутствует (только ручное сохранение) |

## Часто задаваемые вопросы

### Какие календарные приложения поддерживают файлы .ics?

Все основные: Outlook, Apple Calendar, Google Calendar, Thunderbird и любое приложение, поддерживающее стандарт iCalendar.

### Работают ли напоминания во всех календарных приложениях?

Несколько напоминаний работают в Apple Calendar и Thunderbird. Outlook использует только первое напоминание. Google Calendar полностью игнорирует напоминания при импорте.

### Можно ли использовать вебхуки без создания мероприятий Discord?

Да. Вебхук и токен бота — это независимые функции. Вы можете использовать вебхуки для доставки календарей без настройки бота Discord.

### Является ли URL вебхука конфиденциальным?

Да — любой, у кого есть URL вебхука, может отправлять сообщения в этот канал. Обращайтесь с ним как с паролем. Он шифруется и хранится локально с помощью защищённого хранилища вашей операционной системы.

## Устранение неполадок

| Проблема | Решение |
|---|---|
| Файл .ics не генерируется | Убедитесь, что «Включить создание файлов календаря» включено в Настройки > Общие, и «Создать приглашение календаря .ics» отмечено в шаблоне или мероприятии |
| Вебхук не публикует | Проверьте URL вебхука через «Проверить вебхук» в настройках Discord. Убедитесь, что «Отправить .ics в Discord» включено для группы |
| Напоминания не работают в Outlook | Outlook поддерживает только первое напоминание. Приложение располагает самое длительное первым для совместимости |
| Напоминания не работают в Google Calendar | Google Calendar игнорирует пользовательские напоминания при импорте .ics. Настройте напоминания вручную после импорта |
| Автосохранение не работает | Убедитесь, что директория сохранения настроена в Настройки > Информация о приложении |
| Файлы сохраняются в неправильном месте | Файлы сохраняются в `{директория}/{Название группы}/`. Проверьте название группы и путь к директории |
