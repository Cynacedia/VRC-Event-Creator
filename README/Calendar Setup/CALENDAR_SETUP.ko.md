# 캘린더 연동 설정 가이드

이 가이드는 VRC Event Creator의 캘린더 파일(.ics) 생성 및 Discord 웹훅 전달 설정 과정을 안내합니다. 설정이 완료되면 VRChat 이벤트를 생성할 때 캘린더 초대 파일이 자동으로 생성되며, 선택적으로 Discord 채널에 게시할 수 있습니다.

---

## 개요

캘린더 연동은 Outlook, Apple Calendar, Google Calendar 및 기타 캘린더 앱으로 가져올 수 있는 표준 `.ics` 캘린더 파일을 생성합니다. 이 파일에는 이벤트 세부 정보와 선택적 알림이 포함됩니다.

두 가지 전달 방법이 있습니다 (이벤트당 하나만 선택 가능):

- **Discord Webhook** — `.ics` 파일을 Discord 채널에 이벤트 임베드 또는 Discord 이벤트 링크와 함께 게시
- **자동 저장** — `.ics` 파일을 로컬 디렉토리에 자동으로 저장

Discord Webhook이 설정되어 있고 이벤트가 Discord에 게시되도록 설정된 경우 웹훅이 사용됩니다. 그렇지 않으면 파일이 설정된 로컬 디렉토리에 저장됩니다.

---

## 1단계: 캘린더 파일 생성 활성화

1. **설정** > **일반**을 엽니다
2. **"캘린더 파일 생성 활성화"**를 체크합니다

이렇게 하면 템플릿 및 이벤트 생성 시 캘린더 옵션을 사용할 수 있습니다.

## 2단계: 전달 방법 설정

### 옵션 A: Discord Webhook (권장)

웹훅은 캘린더 파일을 특정 Discord 채널에 게시합니다. 웹훅 자체에는 봇이 필요하지 않습니다.

1. Discord에서 캘린더 파일을 게시할 채널을 마우스 오른쪽 버튼으로 클릭합니다
2. **채널 편집** > **연동** > **웹후크** > **새 웹후크**를 클릭합니다
3. 웹훅 URL을 복사합니다
4. VRC Event Creator에서 **설정** > **Discord 연동** > 그룹을 선택합니다
5. **".ics를 Discord에 게시"**를 체크하고 웹훅 URL을 붙여넣습니다
6. **"Webhook 테스트"**를 클릭하여 확인한 후 **"저장"**을 클릭합니다

Discord 이벤트 생성(봇 토큰)도 설정한 경우, 웹훅은 독립적인 임베드 대신 Discord 이벤트 링크를 게시합니다. `.ics` 파일은 어느 경우든 첨부됩니다.

### 옵션 B: 로컬 디렉토리에 자동 저장

웹훅이 설정되지 않은 경우, `.ics` 파일은 로컬 디렉토리에 자동으로 저장됩니다. 기본 위치는 `Documents/VRC Event Creator .ics/`이며 처음 저장할 때 생성됩니다.

파일은 `{디렉토리}/{그룹 이름}/{이벤트 이름 - 날짜}.ics`로 저장됩니다. 위치를 변경하려면 **설정** > **앱 정보**의 **캘린더 저장 디렉토리** 옆에 있는 **변경** 버튼을 사용하세요.

---

## 3단계: 템플릿 설정

1. **템플릿 관리**로 이동하여 템플릿을 편집(또는 생성)합니다
2. **기본** 탭에서 **".ics 캘린더 초대 생성"**을 체크합니다
3. **일정** 탭에 새로운 **".ics 캘린더 알림"** 카드가 나타납니다
4. **".ics 캘린더 알림 활성화"**를 체크하고 원하는 알림 간격을 추가합니다
5. 템플릿을 저장합니다

알림은 모든 주요 캘린더 앱과 호환되는 사전 설정 간격을 사용합니다: 5분, 10분, 15분, 30분, 1시간, 2시간, 4시간, 8시간, 12시간, 1일, 2일, 1주.

> **참고:** 일부 캘린더 앱(Outlook 등)은 첫 번째 알림만 사용합니다. 최대 호환성을 위해 가장 긴 알림이 먼저 배치됩니다. Google Calendar은 가져오기 시 사용자 지정 알림을 무시하고 기본 알림 설정을 사용합니다.

---

## 4단계: 이벤트 생성

이벤트를 생성할 때 (수동 또는 자동화):

- **날짜** 단계에 **".ics 캘린더 초대 생성"** 토글이 표시됩니다 (선택한 템플릿에서 상속되거나 수동으로 설정 가능)
- 아래에 **".ics 캘린더 알림 활성화"**로 이벤트별 알림을 사용자 지정할 수 있습니다
- **세부 정보** 단계에 표시되는 **"Discord에 게시"**가 Discord 이벤트와 웹훅 전달을 모두 제어합니다

템플릿의 모든 설정은 이벤트별로 재정의할 수 있습니다.

---

## 연동 작동 방식

| Discord 이벤트 | Webhook | 캘린더 | 이벤트 생성 시 동작 |
|---|---|---|---|
| 활성화 + 설정됨 | 설정됨 | 활성화 | Discord 이벤트 생성, 웹훅이 이벤트 링크 + .ics 게시 |
| 비활성화 또는 미설정 | 설정됨 | 활성화 | 웹훅이 이벤트 세부 정보 임베드 + .ics 게시 |
| 무관 | 미설정 | 활성화 | .ics 파일이 로컬 디렉토리에 자동 저장 |

## 자주 묻는 질문

### 어떤 캘린더 앱이 .ics 파일을 지원하나요?

모든 주요 캘린더 앱: Outlook, Apple Calendar, Google Calendar, Thunderbird 및 iCalendar 표준을 지원하는 모든 앱.

### 알림이 모든 캘린더 앱에서 작동하나요?

여러 알림은 Apple Calendar과 Thunderbird에서 작동합니다. Outlook은 첫 번째 알림만 사용합니다. Google Calendar은 가져오기 시 알림을 완전히 무시합니다.

### Discord 이벤트 생성 없이 웹훅을 사용할 수 있나요?

네. 웹훅과 봇 토큰은 독립적인 기능입니다. Discord 봇을 설정하지 않고도 웹훅으로 캘린더를 전달할 수 있습니다.

### 웹훅 URL은 민감한 정보인가요?

네 — 웹훅 URL을 가진 사람은 누구나 해당 채널에 메시지를 보낼 수 있습니다. 비밀번호처럼 취급하세요. 운영 체제의 보안 저장소를 사용하여 암호화되며 로컬에 저장됩니다.

## 문제 해결

| 문제 | 해결 방법 |
|---|---|
| .ics 파일이 생성되지 않음 | 설정 > 일반에서 "캘린더 파일 생성 활성화"가 켜져 있는지, 템플릿 또는 이벤트에서 ".ics 캘린더 초대 생성"이 체크되어 있는지 확인하세요 |
| 웹훅이 게시하지 않음 | Discord 설정의 "Webhook 테스트"로 웹훅 URL을 확인하세요. 그룹에서 ".ics를 Discord에 게시"가 활성화되어 있는지 확인하세요 |
| Outlook에서 알림이 작동하지 않음 | Outlook은 첫 번째 알림만 지원합니다. 호환성을 위해 앱이 가장 긴 알림을 먼저 배치합니다 |
| Google Calendar에서 알림이 작동하지 않음 | Google Calendar은 .ics 가져오기 시 사용자 지정 알림을 무시합니다. 가져온 후 수동으로 알림을 설정하세요 |
| 파일이 잘못된 위치에 저장됨 | 파일은 `{저장 디렉토리}/{그룹 이름}/`에 저장됩니다. 기본값은 `Documents/VRC Event Creator .ics/`입니다. 설정 > 앱 정보에서 변경 가능 |
