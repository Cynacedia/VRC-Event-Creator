# 캘린더 연동 설정 가이드

이 가이드는 VRC Event Creator의 캘린더 파일(.ics) 생성, Discord 웹훅 게시, Discord 예약 이벤트 설정 과정을 안내합니다. 이 세 가지 기능은 완전히 독립적입니다 — 워크플로에 맞는 조합을 자유롭게 활성화하세요.

---

## 개요

VRC Event Creator는 VRChat 이벤트를 생성하거나 자동화할 때 세 가지 후속 작업을 제공합니다. 각각은 템플릿별, 이벤트별로 독립적으로 전환할 수 있습니다:

- **".ics 캘린더 초대 생성"** — 선택적 알림이 포함된 표준 `.ics` 캘린더 파일을 생성하고, 로컬 디렉토리에 자동 저장
- **"Discord Webhook 게시"** — 웹훅을 통해 Discord 채널에 공지를 게시 (캘린더도 활성화된 경우 `.ics` 첨부 가능)
- **"Discord 이벤트 생성"** — 봇을 통해 Discord 서버에 예약 이벤트 생성

여러 기능을 활성화하면 자연스럽게 조합됩니다:

| Discord 이벤트 | Webhook | 캘린더 (.ics) | 동작 |
|---|---|---|---|
| 켜짐 | 꺼짐 | 꺼짐 | Discord 예약 이벤트만 생성 |
| 꺼짐 | 켜짐 | 꺼짐 | 웹훅이 이벤트 세부 정보 임베드 게시 |
| 꺼짐 | 꺼짐 | 켜짐 | `.ics` 파일이 로컬 디렉토리에 자동 저장 |
| 켜짐 | 켜짐 | 꺼짐 | Discord 이벤트 생성 + 웹훅이 이벤트 링크 게시 |
| 켜짐 | 꺼짐 | 켜짐 | Discord 이벤트 생성 + `.ics` 자동 저장 |
| 꺼짐 | 켜짐 | 켜짐 | 웹훅이 임베드 + `.ics` 첨부 게시, 로컬에도 저장 |
| 켜짐 | 켜짐 | 켜짐 | Discord 이벤트 + 웹훅이 이벤트 링크 + `.ics` 첨부 + 자동 저장 |

---

## 1단계: 캘린더 파일 생성 활성화

1. **설정** > **고급 설정**을 엽니다
2. **"캘린더 파일 생성 활성화"**를 체크합니다

이렇게 하면 템플릿 및 이벤트 생성 시 **".ics 캘린더 초대 생성"** 토글을 사용할 수 있습니다.

### 자동 저장 디렉토리

캘린더 파일 생성이 활성화되면, `.ics` 파일은 항상 로컬 디렉토리에 저장됩니다. 기본 위치는 `Documents/VRC Event Creator .ics/`이며 처음 저장할 때 생성됩니다.

파일은 `{디렉토리}/{그룹 이름}/{이벤트 이름 - 날짜}.ics`로 저장됩니다. 위치를 변경하려면 **설정** > **앱 정보**의 **캘린더 저장 디렉토리** 옆에 있는 **변경** 버튼을 사용하세요.

---

## 2단계: Discord 웹훅 설정 (선택 사항)

웹훅은 특정 Discord 채널에 공지를 게시합니다. 캘린더 파일 및 Discord 이벤트와 독립적인 기능으로, 어느 것과든 함께 또는 단독으로 사용할 수 있습니다.

1. Discord에서 공지를 게시할 채널을 마우스 오른쪽 버튼으로 클릭합니다
2. **채널 편집** > **연동** > **웹후크** > **새 웹후크**를 클릭합니다
3. 웹훅 URL을 복사합니다
4. VRC Event Creator에서 **설정** > **Discord 연동** > 그룹을 선택합니다
5. **"Webhook 활성화"**를 체크하고 웹훅 URL을 붙여넣습니다
6. **Webhook 테스트**를 클릭하여 확인한 후 **저장**을 클릭합니다

웹훅과 캘린더가 모두 이벤트에 활성화되면, `.ics` 파일이 웹훅 게시물에 첨부됩니다. 웹훅만 활성화된 경우(캘린더 없음), 웹훅은 `.ics` 첨부 없이 이벤트 세부 정보 임베드를 게시합니다.

Discord 예약 이벤트도 생성된 경우, 웹훅 메시지에는 임베드 대신 Discord 이벤트 링크가 포함됩니다.

---

## 3단계: 템플릿 설정

1. **템플릿 관리**로 이동하여 템플릿을 편집(또는 생성)합니다
2. **기본** 탭에서 설정 상황에 따라 최대 세 개의 게시 토글이 표시됩니다:
   - **".ics 캘린더 초대 생성"** — 캘린더 파일 생성이 활성화된 경우 표시
   - **"Discord 이벤트 생성"** — Discord 봇이 그룹에 설정된 경우 표시
   - **"Discord Webhook 게시"** — 웹훅 URL이 그룹에 설정된 경우 표시
3. 이 템플릿에 사용할 것들을 활성화합니다
4. 캘린더가 활성화되면 **일정** 탭에 **".ics 캘린더 알림"** 카드가 표시됩니다
5. **".ics 캘린더 알림 활성화"**를 체크하고 원하는 알림 간격을 추가합니다
6. 템플릿을 저장합니다

알림은 모든 주요 캘린더 앱과 호환되는 사전 설정 간격을 사용합니다: 5분, 10분, 15분, 30분, 1시간, 2시간, 4시간, 8시간, 12시간, 1일, 2일, 1주.

> **참고:** 일부 캘린더 앱(Outlook 등)은 첫 번째 알림만 사용합니다. 최대 호환성을 위해 가장 긴 알림이 먼저 배치됩니다. Google Calendar은 가져오기 시 사용자 지정 알림을 무시하고 기본 알림 설정을 사용합니다.

---

## 4단계: 이벤트 생성

이벤트를 생성할 때 (수동 또는 자동화):

- **날짜** 단계에 **".ics 캘린더 초대 생성"**이 표시됩니다 (템플릿에서 상속, 재정의 가능)
- 아래에 **".ics 캘린더 알림 활성화"**로 이벤트별 알림을 사용자 지정할 수 있습니다
- **세부 정보** 단계에 **"Discord 이벤트 생성"**과 **"Discord Webhook 게시"**가 별도의 토글로 표시됩니다
- 템플릿의 모든 설정은 이벤트별로 재정의할 수 있습니다

## 자주 묻는 질문

### 어떤 캘린더 앱이 .ics 파일을 지원하나요?

모든 주요 캘린더 앱: Outlook, Apple Calendar, Google Calendar, Thunderbird 및 iCalendar 표준을 지원하는 모든 앱.

### 알림이 모든 캘린더 앱에서 작동하나요?

여러 알림은 Apple Calendar과 Thunderbird에서 작동합니다. Outlook은 첫 번째 알림만 사용합니다. Google Calendar은 가져오기 시 알림을 완전히 무시합니다.

### 캘린더 파일 없이 웹훅을 사용할 수 있나요?

네. 웹훅은 캘린더 파일 생성이 비활성화되어 있어도 이벤트 세부 정보 임베드를 게시합니다. ".ics 캘린더 초대 생성"을 활성화하지 않고 템플릿에서 "Discord Webhook 게시"를 활성화하세요.

### Discord 이벤트 생성 없이 웹훅을 사용할 수 있나요?

네. 웹훅, Discord 이벤트, 캘린더 파일은 완전히 독립적입니다. 어떤 조합이든 작동합니다.

### 웹훅 URL은 민감한 정보인가요?

네 — 웹훅 URL을 가진 사람은 누구나 해당 채널에 메시지를 보낼 수 있습니다. 비밀번호처럼 취급하세요. 운영 체제의 보안 저장소를 사용하여 암호화되며 로컬에 저장됩니다.

## 문제 해결

| 문제 | 해결 방법 |
|---|---|
| .ics 파일이 생성되지 않음 | 고급 설정에서 "캘린더 파일 생성 활성화"가 켜져 있는지, 템플릿 또는 이벤트에서 ".ics 캘린더 초대 생성"이 체크되어 있는지 확인하세요 |
| 웹훅이 게시하지 않음 | Discord 설정의 "Webhook 테스트"로 웹훅 URL을 확인하세요. 그룹에서 "Webhook 활성화"가 켜져 있고 템플릿에서 "Discord Webhook 게시"가 체크되어 있는지 확인하세요 |
| 웹훅이 게시되지만 .ics가 첨부되지 않음 | 이벤트에서 ".ics 캘린더 초대 생성"도 활성화해야 합니다. 이것 없이는 웹훅이 임베드 또는 이벤트 링크만 게시합니다 |
| Outlook에서 알림이 작동하지 않음 | Outlook은 첫 번째 알림만 지원합니다. 호환성을 위해 앱이 가장 긴 알림을 먼저 배치합니다 |
| Google Calendar에서 알림이 작동하지 않음 | Google Calendar은 .ics 가져오기 시 사용자 지정 알림을 무시합니다. 가져온 후 수동으로 알림을 설정하세요 |
| 파일이 잘못된 위치에 저장됨 | 파일은 `{저장 디렉토리}/{그룹 이름}/`에 저장됩니다. 기본값은 `Documents/VRC Event Creator .ics/`입니다. 설정 > 앱 정보에서 변경 가능 |
